Since we can't fetch arbitrary experts by username from the backend easily, the simplest approach is: when `savedFilter` is active, the grid should show ALL experts from the `experts` array whose username is in `savedExperts`. BUT the key change is that the parent (MarketplacePage/useExplore) should NOT apply query/tag/rate filters when `savedFilter` is active.
3. Actually, the cleanest approach per the existing architecture: when `savedFilter` is true, pass the full unfiltered expert list. To do this, modify `useExplore.ts` to fetch WITHOUT filters when `savedFilter` is active, OR better yet, keep it simple — the frontend already has `savedExperts` usernames in localStorage. When `savedFilter` toggles ON:
   - Set a flag in the store
   - ExpertGrid/ExpertList reads this flag and shows only cards matching `savedExperts` usernames from whatever is loaded

   The REAL fix for filter-independence: when `savedFilter` is true, `useExplore` should fetch with NO query, NO tags, NO rate filter — just a plain `/api/explore` call that returns all experts, then client-side filter to saved ones.

   Specific approach in `useExplore.ts`:
   - Read `savedFilter` from the store
   - When building the fetch URL params, if `savedFilter` is true: skip query, tags, rateMin, rateMax — fetch with no filters to get all experts
   - The ExpertGrid `displayExperts` memo already filters to saved-only when `savedFilter` is true — this works correctly once we feed it unfiltered data
