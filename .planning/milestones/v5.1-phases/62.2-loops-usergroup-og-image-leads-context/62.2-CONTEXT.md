# Phase 62.2: Loops Usergroup, OG Preview Image, Leads Timeline Context - Context

**Gathered:** 2026-03-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Three targeted improvements:
1. Add `userGroup` field to Loops newsletter API calls
2. Set up OpenGraph and Twitter Card meta tags using `OG picture.png`
3. Link leads to their search sessions via session_id so the leads timeline shows searches alongside clicks

</domain>

<decisions>
## Implementation Decisions

### Loops usergroup
- Send `userGroup: "search"` as a static value on all new Loops API contact creation calls
- New signups only — do not backfill existing contacts
- The Loops API field name is `userGroup`

### OG preview image
- Add OpenGraph meta tags to all pages (not just homepage)
- Use existing `<title>` and description from HTML head — just add the OG image, og:title, og:description
- Also add Twitter Card tags: `twitter:card=summary_large_image` + `twitter:image`
- Claude's Discretion: file placement and naming (rename to URL-friendly name, place in public/)

### Leads timeline — session linking
- Capture the user's current `session_id` when they sign up via the newsletter form
- Send session_id to backend and store it with the lead record
- Current session only — do not attempt to link historical sessions
- Query `user_events` by the linked session_id to retrieve search events for the lead
- Display searches in a unified chronological timeline alongside click events
- Search events show the query text (e.g., "Searched: cybersecurity consultant")

### Claude's Discretion
- OG image file naming and public/ placement
- Exact timeline event styling for search vs click differentiation
- How session_id is passed from frontend to newsletter signup endpoint

</decisions>

<specifics>
## Specific Ideas

- The `OG picture.png` file already exists in the frontend root directory
- Lead timeline currently only shows clicks because `LeadClick` is the only model storing email — searches are in `user_events` with anonymous `session_id`
- The fix requires capturing session_id at signup time to bridge the anonymous search data with the identified lead

</specifics>
