2. Add a private email validation helper function BEFORE the route handler:
   ```python
   def _validate_email(email: str | None) -> str | None:
       """Basic check: must contain @ and a dot. Returns None if invalid."""
       if not email:
           return None
       email = email.strip()
       if "@" not in email or "." not in email:
           return None
       return email.lower()
   ```

3. Update `record_event` handler to validate and store email:
   ```python
   @router.post("/api/events", status_code=202)
   def record_event(body: EventRequest, db: Session = Depends(get_db)):
       validated_email = _validate_email(body.email)
       record = UserEvent(
           session_id=body.session_id,
           event_type=body.event_type,
           payload=json.dumps(body.payload),
           email=validated_email,
       )
       db.add(record)
       db.commit()
       log.info("event.recorded", event_type=body.event_type, session_id=body.session_id, email=validated_email)
       return {"status": "accepted"}
   ```

Key behaviors per CONTEXT.md decisions:
- Events WITHOUT email field still work (email defaults to None) — no regression
- Events WITH invalid email get stored with email=null — never reject/lose a tracking event
- Response stays 202 Accepted with {"status": "accepted"} — no changes to response body

1. In `frontend/src/tracking.ts`, add a helper function to read email from the Zustand persist store:
   ```typescript
   function getSubscriberEmail(): string | null {
     try {
       const raw = localStorage.getItem('tinrate-newsletter-v1')
       if (!raw) return null
       const parsed = JSON.parse(raw)
       return parsed?.state?.email || null
     } catch {
       return null
     }
   }
   ```
   Place it after the existing `getSessionId()` function.

   NOTE on localStorage key: STATE.md decision says "Gate writes exclusively to useNltrStore (tinrate-newsletter-v1) — no new localStorage key." The Zustand persist middleware in `nltrStore.ts` serializes state as `{"state":{"subscribed":true,"email":"user@example.com"},"version":0}` under the `tinrate-newsletter-v1` key. We read from this existing key rather than creating a new `subscriber_email` key.
