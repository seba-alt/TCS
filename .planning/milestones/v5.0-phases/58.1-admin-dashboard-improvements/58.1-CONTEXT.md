# Phase 58.1: Admin Dashboard Improvements - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix bugs and improve the existing admin dashboard — better data presentation, remove broken/irrelevant sections, improve lead click visibility, merge redundant pages, and handle empty states properly. No new admin capabilities — only improve what's already built.

</domain>

<decisions>
## Implementation Decisions

### Overview page cleanup
- Keep both stats cards AND recent activity — combined view serving as "what happened today" + KPIs
- Show tags as chips/badges in Recent Explore Searches rows so tag-based searches aren't empty
- Claude decides which sections to keep/remove/fix (top feedback, top searched, etc.) based on data value analysis
- Claude decides which stat cards to show based on what metrics matter for an expert marketplace admin

### Data page structure
- Merge Searches and Marketplace tabs into a single Data page — the scope doesn't justify the split
- Add a unified date range picker that filters both the trend chart and the search records table together
- Claude decides what content stays (remove duplicates already surfaced on Gaps/Experts pages) and whether sidebar navigation needs restructuring

### Lead activity visibility
- Add a dedicated click activity view — chronological table of all click events, not hidden in expandable rows
- Show click activity on BOTH Overview (recent/summary) and Leads page (full history)
- Each click row shows full context: lead email, expert name, search query that led to the click, timestamp, source (grid/sage)
- Add a click count column to the leads table — visible without expanding, sortable

### Empty state handling
- Sections with no data stay visible with context-specific empty messages (e.g., "No searches in this period", "No click activity yet")
- Tags display as chips/badges (visually distinct from text queries) — matches marketplace filter chip style
- Visually distinguish text-query searches from tag-based explore searches (different icon or label)

### Claude's Discretion
- Which Overview sections to remove vs fix (analyze data value of each)
- Which stat cards to show on Overview
- How to structure the merged Data page (layout, what from Marketplace to keep vs drop)
- Sidebar navigation reorganization if page merges make items redundant
- Specific empty state message copy per section
- Visual design of search type indicators (icons, labels, colors)

</decisions>

- "In data there should be no difference between searches and marketplace — the scope of what we currently have is very clear so it should be reflected in admin"
- "Lot of the explore searches are empty this is due to tags — either show the tags or do not show row at all"
- "Top feedback is irrelevant, top searched is empty"
- "I still cannot see what leads clicked who" — need click activity to be immediately visible, not buried in expandable rows
- Research best practices for this kind of admin/analytics dashboard
