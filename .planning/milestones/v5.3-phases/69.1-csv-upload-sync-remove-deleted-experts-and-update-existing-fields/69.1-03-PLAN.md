  <name>Task 1: Redesign CsvImportModal with sync preview and cherry-pick UI</name>
  <files>frontend/src/admin/components/CsvImportModal.tsx</files>
  <action>
Rewrite CsvImportModal.tsx to support the new sync flow. The new flow is:

**Step flow:** idle -> preview -> mapping -> syncing-preview -> sync-review -> applying -> done -> error

**State additions:**
- Add `syncPlan` state: `{ to_add: [], to_update: [], to_delete: [], summary: {} }`
- Add `skipDeletes` state: `Set<string>` — usernames unchecked from deletion list
- Update `Step` type to add: `'syncing-preview' | 'sync-review' | 'applying'`
- Update `ImportResult` interface to: `{ inserted: number, updated: number, deleted: number, reactivated: number, skipped: number, rebuilding: boolean }`

**Updated flow after column mapping step:**

Replace the old `handleImport()` with two functions:

1. `handleSyncPreview()` — called when user clicks "Preview Sync" button (replaces old "Import" button):
   - Sets step to `'syncing-preview'` (shows spinner)
   - POSTs to `/api/admin/experts/sync-preview` with file + column_mapping (same FormData as before)
   - On success, stores response in `syncPlan` state, initializes `skipDeletes` as empty Set, sets step to `'sync-review'`

2. `handleSyncApply()` — called when user clicks "Apply Sync" button from review screen:
   - Sets step to `'applying'` (shows spinner)
   - POSTs to `/api/admin/experts/sync-apply` with file + column_mapping + skip_deletes (JSON array from skipDeletes Set)
   - On success, stores result, sets step to `'done'`
   - Does NOT trigger a separate FAISS rebuild — the backend does it automatically

**New "sync-review" step UI** (between mapping and applying):

Render a scrollable panel with three collapsible sections:

**Section 1: "Experts to Add" (green accent)**
- Header: "N expert(s) to add" with green plus icon
- Collapsed by default if empty, expanded if non-empty
- List each expert: `{first_name} {last_name} — {job_title}`

**Section 2: "Experts to Update" (blue accent)**
- Header: "N expert(s) to update" with blue pencil icon
- Show reactivation count if any: "(including N reactivations)"
- List each expert with name, and a small "Changes:" sub-list showing changed fields
- For reactivating experts, show a green "Reactivating" badge

**Section 3: "Experts to Remove" (red/amber accent)**
- Header: "N expert(s) to remove" with red trash icon
- Each expert has a checkbox (checked by default = will be deleted)
- Admin can uncheck to skip deletion for specific experts
- List each expert: checkbox + `{first_name} {last_name} — {job_title}`
- Unchecked experts are added to `skipDeletes` Set
- Toggle handler: `toggleSkipDelete(username)` — adds/removes from skipDeletes Set

**Summary bar** at the bottom of sync-review:
- "N adds, N updates, N removals" with the removal count adjusted for skipped ones
- If any experts unchecked: "(N experts kept)"

**Buttons:**
- "Back" (goes to mapping step)
- "Apply Sync" (calls handleSyncApply)

**Updated "done" step:**
- Show: `{inserted} added, {updated} updated, {deleted} removed, {reactivated} reactivated`
- Show "FAISS rebuild started in background" if `rebuilding` is true
- "Close" button

**Styling:** Follow existing admin dark theme (slate-800/700/900 backgrounds, purple accents, same button styles). Use Tailwind classes consistent with the existing modal.

**The "syncing-preview" spinner step** should say "Analyzing sync changes..." instead of "Importing experts..."

**The "applying" spinner step** should say "Applying sync..."

Keep the old preview-csv call for the initial preview (step 1-2 remain unchanged — shows first 5 rows). The sync-preview is a separate call that happens after column mapping.
