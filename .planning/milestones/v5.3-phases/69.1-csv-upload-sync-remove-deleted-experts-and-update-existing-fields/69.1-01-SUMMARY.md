## Files Created/Modified

- `app/services/explorer.py` - Added `is_active == True` filter in saved-view shortcut and Stage 1 pre-filter
- `app/routers/browse.py` - Added `is_active == True` to all 7 Expert queries (category counts, category experts, recently-added total, recently-added list, cold-start total, cold-start list, featured)

## Decisions Made
- Soft-delete over hard-delete: Expert records preserved for historical data and analytics; `is_active=False` hides from all public interfaces
- Photo proxy endpoint intentionally left unfiltered — admin can still proxy photos by username for management purposes
- Removed metadata.json and experts.csv cleanup from delete endpoints — FAISS rebuild handles exclusion via the `is_active` filter in `load_tagged_experts()`, simplifying delete logic significantly
- `server_default="1"` used alongside `default=True` for SQLite compatibility — existing rows get `is_active=1` without a migration script
