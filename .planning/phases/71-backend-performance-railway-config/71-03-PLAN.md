Changes from current state:
- `startCommand`: Added `--timeout-keep-alive 75` (keep TCP connections alive longer than Railway's 60s load balancer timeout — prevents premature connection drops), `--log-level warning` (suppress INFO noise in production), `--no-access-log` (Uvicorn access logs are redundant with structlog request logging)
- `healthcheckTimeout`: Reduced from 300 to 120 (seconds). Railway will mark the service unhealthy if `/api/health` does not respond within 120s of deployment start.
- `restartPolicyType`: `ON_FAILURE` — Railway restarts the container automatically if the process exits with a non-zero code. Does NOT restart on successful exit.
- `region`: `europe-west4` (Netherlands) — closest Railway region to primary European audience.
