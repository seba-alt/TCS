**app/main.py — Event flush worker in lifespan + GZipMiddleware:**

1. Add imports: `import asyncio`, `from app.event_queue import _event_queue`, `from sqlalchemy import text`
2. Add `_event_flush_worker()` async function at module level (before the lifespan):
   - Constant: `FLUSH_INTERVAL = 2.0`, `BATCH_SIZE = 10`
   - Infinite loop:
     - Collect batch: loop up to BATCH_SIZE items, using `asyncio.wait_for(_event_queue.get(), timeout=remaining)` with a deadline of `FLUSH_INTERVAL` seconds
     - If batch is non-empty, flush:
       - Two attempts (retry once):
         - `with SessionLocal() as db:` → `db.execute(text("INSERT INTO user_events (session_id, event_type, payload, email) VALUES (:s, :e, :p, :m)"), batch_params)` → `db.commit()`
         - If first attempt fails, retry once. If second attempt fails, log error with `structlog` and drop the batch.
3. In the `lifespan` function, after all existing startup code but before `yield`:
   - `flush_task = asyncio.create_task(_event_flush_worker())`
4. After `yield` (shutdown), cancel the flush task:
   - `flush_task.cancel()`
   - `try: await flush_task` / `except asyncio.CancelledError: pass`
5. Add GZipMiddleware AFTER the CORSMiddleware registration:
   - `from fastapi.middleware.gzip import GZipMiddleware`
   - `app.add_middleware(GZipMiddleware, minimum_size=500)`
   - Place this line AFTER the `app.add_middleware(CORSMiddleware, ...)` block (Starlette applies middlewares in LIFO order, so GZip registered after CORS means GZip runs on the response after CORS headers are set)

**app/services/explore_cache.py — New TTL cache module (follows project pattern from _embed_cache and _settings_cache):**

Create new file with:
- Module-level dict `_cache: dict[str, tuple[Any, float]] = {}` (key → (value, timestamp))
- Thread lock `_cache_lock = threading.Lock()` (explore runs in thread executor)
- Constants: `EXPLORE_CACHE_TTL = 300.0` (5 minutes per user decision in CONTEXT.md — user said "5-minute TTL for LRU", overrides the 30s in BPERF-07), `EXPLORE_CACHE_MAX_SIZE = 200`
- `get_cached(key: str) -> Any | None`: Check if key exists and not expired. If expired, delete and return None.
- `set_cached(key: str, value: Any) -> None`: If at max capacity, evict oldest entry. Store `(value, time.time())`.
- `invalidate_explore_cache() -> None`: Clear the entire `_cache` dict.

All operations use `_cache_lock` for thread safety.

**app/routers/explore.py — Add caching:**

1. Import `from app.services.explore_cache import get_cached, set_cached`
2. Before the `run_in_executor` call, construct a cache key:
   - `cache_key = f"{query}|{rate_min}|{rate_max}|{sorted(tag_list)}|{sorted(industry_tag_list)}|{limit}|{cursor}|{seed}|{sorted(username_list) if username_list else ''}"`
3. Check cache: `cached = get_cached(cache_key)` — if not None, return it directly (skip executor)
4. After getting the result from `run_in_executor`, store it: `set_cached(cache_key, result)`
5. Do NOT cache when `seed > 0` — seeded results are randomized, caching defeats the purpose. Only cache when `seed == 0` (deterministic sort).
