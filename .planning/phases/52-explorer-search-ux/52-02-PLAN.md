**Backend suggest improvements (EXPL-05):**

In `app/routers/suggest.py`:
1. Add a `tags` column to the FTS5 query loop. Currently `_run_suggest_multi` queries `job_title` and `company` columns. Add `tags` as the FIRST column in the iteration order: `for column in ('tags', 'job_title', 'company')`. This means tag matches are fetched first and appear before job title/company matches in the deduplication.
2. The FTS5 virtual table already has a `tags` column (confirmed in `app/main.py` — the FTS5 table indexes `first_name, last_name, job_title, company, bio, tags`). The tags column contains JSON arrays like `["seo","marketing"]`, so FTS5 will match on individual tag words within the JSON.
3. Increase the per-column limit from 3 to 5 for the tags column to get more tag-based results. Keep job_title and company at 3. Adjust the loop:
   ```python
   for column, col_limit in [('tags', 5), ('job_title', 3), ('company', 3)]:
       ...
       f"LIMIT {col_limit}"
   ```
4. For the tags column, the DISTINCT query will return raw JSON strings. Post-process tag results: parse individual tag values from the JSON array strings and match them against the query prefix. Extract individual tags that start with or contain the query substring and add those as suggestions (not the raw JSON array string).

   Specifically, for the `tags` column, instead of using the raw FTS5 DISTINCT result, do:
   ```python
   if column == 'tags':
       # Parse individual tags from JSON array strings returned by FTS5
       for row in rows:
           raw = row[0]
           if not raw:
               continue
           try:
               tag_list = json.loads(raw) if raw.startswith('[') else [raw]
           except (json.JSONDecodeError, TypeError):
               tag_list = [raw]
           for tag in tag_list:
               tag = tag.strip()
               if tag and q.strip().lower() in tag.lower():
                   results.append(tag)
       continue  # skip the normal col_results.extend
   ```
5. Add `import json` at the top of the file.
