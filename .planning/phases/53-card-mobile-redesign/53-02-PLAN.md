Modify both `toggleTag` and `toggleIndustryTag` in filterSlice.ts to also clear the search query when a tag is toggled ON (added). When toggling OFF (removing), do NOT clear the query — only clear on add.

**toggleTag change:**
```typescript
toggleTag: (tag) => set((state) => {
  const isRemoving = state.tags.includes(tag)
  return {
    tags: isRemoving
      ? state.tags.filter((t) => t !== tag)
      : [...state.tags, tag],
    // Clear search query when adding a tag (per CONTEXT.md: clicking a tag resets search)
    ...(isRemoving ? {} : { query: '' }),
  }
}),
```

**toggleIndustryTag change:**
Same pattern — clear query only when adding an industry tag:
```typescript
toggleIndustryTag: (tag) => set((state) => {
  const isRemoving = state.industryTags.includes(tag)
  return {
    industryTags: isRemoving
      ? state.industryTags.filter((t) => t !== tag)
      : [...state.industryTags, tag],
    ...(isRemoving ? {} : { query: '' }),
  }
}),
```

This approach ensures the query-clearing behavior applies everywhere toggleTag is called (ExpertCard tag pills, TagCloud, MobileInlineFilters picker, FilterSidebar TagMultiSelect) without needing to modify each call site.

NOTE: Per CONTEXT.md decision — "Clicking a tag resets the active search query (clears the search bar) on BOTH mobile and desktop" and "Clicking a tag adds to existing active tags (does not replace them)". This implementation achieves both: query is cleared, but tags accumulate.
