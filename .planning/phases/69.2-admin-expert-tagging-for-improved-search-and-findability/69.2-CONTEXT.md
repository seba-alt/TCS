# Phase 69.2: Admin Expert Tagging for Improved Search and Findability - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a dedicated admin tag manager that lets admins create domain expertise tags and assign them to experts (both directions: expert→tags and tag→experts). Manual tags improve search findability and are browsable/filterable by public users. Tags are additive only — admins cannot remove AI-generated tags.

</domain>

<decisions>
## Implementation Decisions

### Tag editing UI
- Dedicated tag manager page in admin panel (not inline on expert detail)
- Two navigation directions: select expert → add tags, OR select tag → assign to experts
- Predefined tag list only — admin must explicitly create tags before assigning them (no free-text autocomplete)
- Bulk-tag support: select multiple experts and apply one or more tags to all at once

### Tag types & scope
- Admin creates domain expertise tags (e.g., "machine learning", "tax law") — NOT organizational labels
- Admin can only ADD manual tags, not remove auto-generated AI (Gemini skill) or industry tags
- Predefined tag list is seeded from existing unique AI-generated skill tags as the starting list
- Admin can add new tags to the predefined list as needed

### AI vs manual coexistence
- Manual tags always survive AI re-tagging — Gemini only touches its own tags
- No visual distinction between AI and manual tags — all tags look the same
- Tags are deduplicated: if AI and admin both add the same tag, it appears once
- Manual tags display on public-facing expert profiles/cards alongside AI tags

### Search & findability
- Tags are browsable/filterable by users on the public site (clickable)
- FAISS index rebuild is manual after tag changes (admin triggers when ready), NOT automatic

### Claude's Discretion
- Data model approach: new "manual" tag type vs editing existing skill tags
- AI re-tagging merge strategy details (manual tags always survive is locked)
- Search weight: whether manual tags get a relevance boost over AI tags
- Tag click behavior on public site: filter browse page vs trigger search
- Tag manager page layout and interaction details
- How to handle tag deduplication at the data layer

</decisions>
