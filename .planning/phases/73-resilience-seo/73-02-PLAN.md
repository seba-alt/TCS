<task type="auto">
  <name>Task 2: Create robots.txt and sitemap.xml in public directory</name>
  <files>frontend/public/robots.txt, frontend/public/sitemap.xml</files>
  <action>
1. Create `frontend/public/robots.txt` with the following content:
   ```
   User-agent: *
   Allow: /
   Disallow: /admin

   Sitemap: https://tcs-three-sigma.vercel.app/sitemap.xml
   ```

   Notes:
   - `Disallow: /admin` blocks all admin routes from crawlers (per SEO-02)
   - `Sitemap` directive helps crawlers discover the sitemap (standard practice)
   - Vercel serves static files from `public/` before the SPA rewrite in vercel.json, so `/robots.txt` resolves correctly
   - No trailing newline after the Sitemap line

2. Create `frontend/public/sitemap.xml` with the following content:
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <url>
       <loc>https://tcs-three-sigma.vercel.app/</loc>
       <lastmod>2026-03-05</lastmod>
       <changefreq>weekly</changefreq>
       <priority>1.0</priority>
     </url>
   </urlset>
   ```

   Notes:
   - Single URL entry — the app is a single-page application with one public route
   - `lastmod` set to current date (2026-03-05) — acceptable for launch; dynamic generation deferred to SEOA-* future requirements
   - `changefreq` and `priority` are optional but standard practice
   - Vercel serves this from `public/` before the SPA rewrite, same as robots.txt
  </action>
  <verify>
    <automated>test -f frontend/public/robots.txt && grep -q "Disallow: /admin" frontend/public/robots.txt && test -f frontend/public/sitemap.xml && grep -q "lastmod" frontend/public/sitemap.xml && echo "PASS: robots.txt and sitemap.xml exist with correct content" || echo "FAIL: missing files or content"</automated>
    <manual>Check that robots.txt contains Disallow: /admin and sitemap.xml contains a lastmod date</manual>
  </verify>
  <done>robots.txt serves Disallow: /admin with Sitemap directive. sitemap.xml lists root URL with 2026-03-05 lastmod date. Both files are in frontend/public/ where Vite copies them to dist/ and Vercel serves them before the SPA rewrite.</done>
</task>
