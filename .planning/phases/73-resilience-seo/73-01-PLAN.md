  <action>
**In `frontend/src/main.tsx`:**

1. Add global `unhandledrejection` handler BEFORE `createRoot()` call (after imports, before router definition):
   ```typescript
   // RSIL-02: Catch unhandled promise rejections to prevent silent blank screens.
   // Sentry's browserTracingIntegration already captures these; this ensures they
   // are logged to console and never silently swallowed.
   window.addEventListener('unhandledrejection', (event) => {
     console.error('[TCS] Unhandled promise rejection:', event.reason)
   })
   ```
   Important: Do NOT call `event.preventDefault()` — Sentry must still see the rejection.

2. Import `ErrorBoundary` from `react-error-boundary` and `ErrorFallback` from `./components/ErrorFallback`

3. Import `* as Sentry` from `@sentry/react` (already available, used in instrument.ts)

4. Wrap `<RouterProvider router={router} />` inside `<StrictMode>` with an `<ErrorBoundary>`:
   ```tsx
   <StrictMode>
     <ErrorBoundary
       fallbackRender={(props) => <ErrorFallback {...props} />}
       onError={(error, info) => {
         Sentry.captureException(error, { extra: { componentStack: info.componentStack } })
       }}
       onReset={() => {
         window.location.href = '/'
       }}
     >
       <RouterProvider router={router} />
     </ErrorBoundary>
   </StrictMode>
   ```
   The `onReset` navigates to root — safest recovery for a catastrophic crash at app level.

**In `frontend/src/pages/MarketplacePage.tsx`:**

1. Import `ErrorBoundary` from `react-error-boundary` and `ErrorFallback` from `../components/ErrorFallback`

2. Import `* as Sentry` from `@sentry/react`

3. Wrap the main content (everything inside the return's `<AuroraBackground>`) with an `<ErrorBoundary>`:
   - Place the boundary INSIDE `<AuroraBackground>` wrapping the `<div className="flex flex-col h-screen overflow-x-hidden">` and all its children
   - Use `fallbackRender={(props) => <ErrorFallback {...props} />}`
   - Use `onError={(error, info) => { Sentry.captureException(error, { extra: { componentStack: info.componentStack } }) }}`
   - Use `onReset={() => { window.location.reload() }}` — page-level recovery reloads the Explorer
   - This boundary catches ExpertGrid render errors, filter errors, header errors — anything inside MarketplacePage

Note: Do NOT add a separate boundary around ExpertGrid — it already has its own inline error handling for API failures. The MarketplacePage-level boundary catches render crashes that the inline handler cannot.
