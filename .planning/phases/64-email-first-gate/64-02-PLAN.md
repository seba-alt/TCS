Modify `get_lead_timeline()` in `app/routers/admin/leads.py` to include email-attributed events from the `user_events` table.

**Current behavior:** The endpoint fetches events from 3 sources:
1. `conversations` table (Sage/chat searches) — matched by email
2. `lead_clicks` table (expert card clicks from lead tracking) — matched by email
3. `user_events` table — matched by session_id (anonymous, linked via newsletter_subscriber.session_id)

**New behavior:** Add a 4th source — email-attributed `user_events`:
- Query `user_events` WHERE `email = :email` AND `event_type IN ('search_query', 'card_click')`
- This captures Explorer search queries and card clicks fired AFTER gate submission (when trackEvent includes email)
- These are DISTINCT from the session_id-linked events (source #3) — session_id events are pre-gate anonymous activity, email events are post-gate identified activity

**Implementation:**

After the existing session event query block (step 3), add step 3.5:

```python
# 3.5. Fetch email-attributed user_events (post-gate activity — TRACK-03)
email_ue_rows = db.scalars(
    select(UserEvent).where(
        UserEvent.email == email,
        UserEvent.event_type.in_(["search_query", "card_click"]),
    )
).all()

for row in email_ue_rows:
    payload_data = json.loads(row.payload or "{}")
    if row.event_type == "search_query":
        search_events.append({
            "type": "explorer_search",
            "query": payload_data.get("query_text", ""),
            "result_count": payload_data.get("result_count", 0),
            "created_at": row.created_at.isoformat(),
        })
    elif row.event_type == "card_click":
        click_events.append({
            "type": "explorer_click",
            "expert_username": payload_data.get("expert", ""),
            "expert_name": payload_data.get("expert", ""),
            "search_query": None,
            "created_at": row.created_at.isoformat(),
        })
```

**Key decisions:**
- Use `"explorer_search"` and `"explorer_click"` as distinct type values (not `"search"` / `"click"`) to differentiate from Sage searches and lead_click entries in the timeline UI
- Match by email ONLY (per CONTEXT.md: "Lead events matched by email only — no session_id fallback")
- The existing session_id-based query (step 3) stays — it captures pre-gate anonymous activity linked via newsletter signup session_id. The new email query captures post-gate identified activity.

**Deduplication note:** There is potential for overlap between step 3 (session_id match) and step 3.5 (email match) if a `search_query` event has BOTH a matching session_id AND email. To prevent duplicates, collect the IDs of events already fetched in step 3, and exclude them in step 3.5:

```python
# Track IDs from session-linked events to avoid duplicates
session_event_ids = {row.id for row in session_event_rows} if subscriber and getattr(subscriber, 'session_id', None) else set()

# In step 3.5, filter:
for row in email_ue_rows:
    if row.id in session_event_ids:
        continue  # Already included via session_id match
    # ... rest of processing
```
