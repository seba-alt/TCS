<objective>
Add in-memory TTL caches for query embeddings (PERF-01) and intelligence settings (PERF-04) to eliminate redundant Google API calls and database reads on repeated requests.

Purpose: Repeated identical search queries currently trigger a ~500ms Google embedding API call every time. Intelligence settings are read from the database on every chat message. Both are stateless lookups that change rarely and benefit from short-lived caches.

Output: Modified `embedder.py` with 60s TTL embedding cache; modified `search_intelligence.py` with 30s TTL settings cache and an `invalidate_settings_cache()` function for the admin settings endpoint to call after writes.
</objective>
