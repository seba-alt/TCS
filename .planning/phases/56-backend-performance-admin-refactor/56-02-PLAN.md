<objective>
Replace LIKE-on-JSON tag filtering with an indexed join table (PERF-02) and pre-fetch feedback data once per explore request (PERF-03) to eliminate the two biggest performance bottlenecks in the explore pipeline.

Purpose: Tag filtering currently uses `Expert.tags.like(f'%"{tag}"%')` which is 55x slower than an indexed join (0.276ms vs 0.005ms per query). Feedback is re-fetched from the database inside the scoring loop on every explore call. Both are unnecessary work.

Output: New `ExpertTag` model, `tag_sync.py` helper module, modified `explorer.py` with JOIN-based tag filtering and request-scoped feedback prefetch, modified `main.py` with startup population.
</objective>

**Step 1: Replace LIKE tag filtering in explorer.py**

Find all occurrences of `Expert.tags.like(f'%"{tag...}"%')` or similar LIKE patterns for tag filtering in `explorer.py` (research notes: lines ~221, ~225). Replace with `exists()` subquery using ExpertTag:

```python
from sqlalchemy import exists
from app.models import ExpertTag

# For skill tags:
stmt = stmt.where(
    exists().where(
        ExpertTag.expert_id == Expert.id,
        ExpertTag.tag == tag.lower(),
        ExpertTag.tag_type == "skill",
    )
)

# For industry tags (if filtering by industry_tags exists):
stmt = stmt.where(
    exists().where(
        ExpertTag.expert_id == Expert.id,
        ExpertTag.tag == industry_tag,
        ExpertTag.tag_type == "industry",
    )
)
```

Each tag in the filter list gets its own `exists()` clause (AND logic — expert must have ALL selected tags). This matches the existing LIKE behavior where multiple tags are AND-ed.

Do NOT remove the `Expert.tags` or `Expert.industry_tags` columns — they remain for serialization/export.

**Step 2: Pre-fetch feedback in explorer.py**

Find the feedback fetch in `run_explore()` (research notes: line ~317, `select(Feedback).where(Feedback.vote.in_(["up", "down"]))`). Move it to the top of the function, before the scoring loop:

1. At the top of `run_explore()`, after determining `is_text_query`, pre-fetch feedback rows:
   ```python
   feedback_rows = []
   if is_text_query:
       try:
           feedback_rows = db.scalars(
               select(Feedback).where(Feedback.vote.in_(["up", "down"]))
           ).all()
       except Exception as exc:
           log.warning("explore.feedback_prefetch_failed", error=str(exc))
   ```

2. Pass `feedback_rows` into wherever the scoring/feedback-boost logic currently fetches from DB. Remove the original fetch that happens inside the scoring section.

The key change: feedback is fetched ONCE at the top, not conditionally inside the scoring loop. The `feedback_rows` list is passed into the scoring step instead of the scoring step doing its own DB query.
