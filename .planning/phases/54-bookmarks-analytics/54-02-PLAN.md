Per ANLT-02 and CONTEXT.md decision: integrate Clarity with project ID vph5o95n6c. The user provided the exact script.

**Implementation approach:** Add Clarity script directly to `index.html` `<head>` section, next to the existing GA4 script. This is the simplest approach (per Claude's discretion on injection method). Since admin routes are within the same SPA, we need to handle exclusion.

**For admin exclusion:** Clarity's script will load on all pages since it's in index.html. However, since admin is behind authentication and represents a tiny fraction of traffic, the simplest approach is to accept that Clarity loads everywhere. If strict exclusion is needed, we could conditionally init Clarity in a React component — but per CONTEXT.md "Active on all public pages (Explorer), exclude admin routes", the cleanest way with index.html injection is to add a small inline script that checks `window.location.pathname` before initializing:

Add after the GA4 script block in `<head>`:

```html
<!-- Microsoft Clarity -->
<script>
  (function(c,l,a,r,i,t,y){
    // Skip Clarity on admin routes
    if (window.location.pathname.startsWith('/admin')) return;
    c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
    t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
    y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
  })(window, document, "clarity", "script", "vph5o95n6c");
</script>
```

This uses the exact user-provided script but wraps it with an early return for `/admin` paths. The check happens at page load time, which works because admin routes all start with `/admin`.

Do NOT modify any other files. Do NOT add npm dependencies.
