**Backend — app/routers/events.py:**

Add a `BatchEventRequest` model and `POST /api/events/batch` endpoint alongside the existing single-event endpoint:

```python
class BatchEventRequest(BaseModel):
    events: list[EventRequest] = Field(..., min_length=1, max_length=50)

@router.post("/api/events/batch", status_code=202)
async def record_event_batch(body: BatchEventRequest):
    for event in body.events:
        validated_email = _validate_email(event.email)
        item = {
            "session_id": event.session_id,
            "event_type": event.event_type,
            "payload": json.dumps(event.payload),
            "email": validated_email,
        }
        try:
            _event_queue.put_nowait(item)
        except asyncio.QueueFull:
            try:
                _event_queue.get_nowait()
            except asyncio.QueueEmpty:
                pass
            _event_queue.put_nowait(item)
    return {"status": "accepted", "count": len(body.events)}
```

Keep the existing single-event endpoint unchanged (backward compatibility).

**Frontend — frontend/src/tracking.ts:**

Replace the current fire-and-forget individual POST pattern with a module-level queue:

1. Add module-level state: `const queue: Array<{session_id, event_type, payload, email}>` initialized as empty array, `let timer: ReturnType<typeof setTimeout> | null = null`.
2. Constants: `BATCH_SIZE = 10`, `FLUSH_INTERVAL_MS = 3000`.
3. `trackEvent()` pushes an event item onto the queue (with session_id, event_type, payload, email resolved at call time). If `queue.length >= BATCH_SIZE`, call `flush()`. Otherwise if no timer is running, start one: `timer = setTimeout(flush, FLUSH_INTERVAL_MS)`.
4. `flush()` function: Clear timer if running. If queue is empty, return. Splice all items from queue (`const batch = queue.splice(0)`). Fire `void fetch(API_BASE + '/api/events/batch', { method: 'POST', headers: {'Content-Type': 'application/json'}, keepalive: true, body: JSON.stringify({ events: batch }) })`. On error, silently drop (Phase 74 handles retry/offline). Export `flush` for testing.
5. Register `window.addEventListener('beforeunload', flush)` at module level (guard with `typeof window !== 'undefined'`).
6. Keep the existing `EventType` type, `TrackPayload` interface, `getSessionId()`, and `getSubscriberEmail()` functions unchanged.

**frontend/vite.config.ts:**

Expand the existing `manualChunks` function inside `build.rollupOptions.output` to add five new vendor chunks. Keep existing recharts and tanstack-table entries. Add:

```typescript
manualChunks(id) {
  if (id.includes('node_modules')) {
    if (id.includes('recharts'))              return 'vendor-charts'
    if (id.includes('@tanstack/react-table')) return 'vendor-table'
    if (id.includes('motion'))                return 'vendor-motion'
    if (id.includes('react-virtuoso'))        return 'vendor-virtuoso'
    if (id.includes('lucide-react'))          return 'vendor-icons'
    if (id.includes('react-use-intercom'))    return 'vendor-intercom'
    if (id.includes('react-router'))          return 'vendor-router'
  }
}
```

Order: most specific first (packages with `@` scopes), then general patterns. The `motion` match is intentionally broad to capture the `motion` package (not `framer-motion`).
