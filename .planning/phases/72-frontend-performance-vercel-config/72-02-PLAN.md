  <action>
Add a `headers` array to the existing `frontend/vercel.json`. Keep the existing `redirects` and `rewrites` arrays unchanged. The final file should be:

```json
{
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    },
    {
      "source": "/(.*\\.(?:png|jpg|jpeg|gif|svg|ico|webp))",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=86400, stale-while-revalidate=604800"
        }
      ]
    }
  ],
  "redirects": [
    { "source": "/marketplace", "destination": "/explore", "permanent": true }
  ],
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**Key details:**
- `/assets/(.*)` matches Vite's hashed output files (JS, CSS). The `immutable` directive tells browsers these never change (content-hash in filename handles cache busting on new deploys). `max-age=31536000` = 1 year.
- The image pattern `/(.*\.(?:png|jpg|jpeg|gif|svg|ico|webp))` matches static images served from the public directory root (e.g., `/og-image.png`, `/favicon.png`). `max-age=86400` = 24 hours, `stale-while-revalidate=604800` = 7 days (browser can serve stale while revalidating in background).
- `headers` array comes first in the JSON for readability (processed before redirects/rewrites by Vercel).
- The image pattern does NOT match `/assets/` images (those are already covered by the first rule with immutable caching).
